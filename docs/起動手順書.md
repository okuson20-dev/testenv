# Network Lab 起動手順書

本書は、**初めて環境を構築してからコンテナが起動するまで**の手順です。  
日常的な設定変更やコマンド一覧は [README.md](../README.md) を参照してください。

---

## 1. 完成イメージ

起動が成功すると、次のコンテナが動作します。

| コンテナ名 | 役割 | 例（デフォルト設定） |
|------------|------|----------------------|
| `lab-router` | セグメント間ルーティング | 各セグメント `.1` |
| `lab-ntp-primary` / `lab-ntp-secondary` | NTP | `172.30.99.10` / `172.30.10.10` |
| `lab-dns-primary` / `lab-dns-secondary` | DNS | `172.30.10.53` / `172.30.20.53` |
| `lab-dhcp-main` | DHCP | `172.30.20.2` |
| `lab-syslog-main` | Syslog | `172.30.99.14` |
| `lab-snmp-main` | SNMP | `172.30.99.161` |

```
[ Windows PC + Docker Desktop ]
           │
    ┌──────┴──────┐
    │ lab-router  │
    └──┬────┬─────┘
  seg-A │    │ seg-mgmt / seg-B
        └────┴─ 各サービス
```

## 2. 必要なもの

| 項目 | バージョン・備考 |
|------|------------------|
| OS | Windows 10/11（本手順の主対象）、macOS / Linux も可 |
| Docker Desktop | 最新版推奨。**WSL2 バックエンド**を有効にする |
| Python | 3.9 以上（`generate` 用。追加パッケージは JSON 設定なら不要） |
| LAN（任意） | 物理 NIC の GW 自動検出を使う場合、LAN ケーブル接続済み |

### 2.1 事前確認コマンド

**PowerShell** で実行し、すべて表示されれば OK です。

```powershell
docker --version
docker compose version
python --version
# または
py -3 --version
```

`python` が無い場合は [python.org](https://www.python.org/downloads/) からインストールし、  
インストーラで **「Add python.exe to PATH」** にチェックを入れてください。

> **`lab.ps1 up` で「Python」とだけ出て止まる場合**  
> Store 用スタブだけが入っていることがあります。  
> **設定 → アプリ実行エイリアス** で `python.exe` をオフし、`py -3 --version` が通ることを確認してから再実行してください。

## 3. ソースの取得

### 方法 A: Git で clone（推奨・複数 PC で最短）

リモート URL は社内 GitHub / GitLab 等に合わせて読み替えてください。

```powershell
cd C:\
git clone https://github.com/okuson20-dev/testenv.git
cd testenv
```

以降の更新:

```powershell
cd C:\testenv
git pull
.\scripts\lab.ps1 restart
```

| メリット | 説明 |
|----------|------|
| 配布が速い | ZIP 配布・手動コピーが不要 |
| 版が揃う | タグ・ブランチで環境を固定できる |
| 設定だけローカル | `config\lab.json` は gitignore のため pull しても上書きされない |

### 方法 B: ZIP / フォルダコピー

Git を使わない場合は、プロジェクトフォルダ一式をコピーして `C:\network-lab` 等に配置します。

## 4. 初回セットアップ（Windows）

### ステップ 4-1: Docker Desktop のインストールと起動

1. [Docker Desktop for Windows](https://www.docker.com/products/docker-desktop/) をインストール
2. 設定 → **General** → **Use the WSL 2 based engine** をオン
3. Docker Desktop を起動し、ステータスが **Running** になるまで待つ

### ステップ 4-2: プロジェクトディレクトリへ移動

clone 済み、またはコピー済みのパスへ移動します（例: `C:\network-lab`）。

```powershell
cd C:\testenv
dir
```

次のフォルダが見えることを確認します。

- `config\`
- `scripts\`
- `services\`
- `README.md`

### ステップ 4-3: 設定ファイルの作成

**JSON（推奨・追加ライブラリ不要）:**

```powershell
copy config\lab.json.example config\lab.json
```

**YAML を使う場合:**

```powershell
copy config\lab.yaml.example config\lab.yaml
pip install -r requirements.txt
```

> `config\lab.json` と `config\lab.yaml` の両方がある場合、**YAML が優先**されます。

### ステップ 4-4: 設定のカスタマイズ（任意）

`config\lab.json`（または `lab.yaml`）をエディタで開き、必要に応じて変更します。

| 設定 | 説明 |
|------|------|
| `services.*.enabled` | `false` でそのサービスを起動しない |
| `services.*.ip` | コンテナの固定 IP |
| `services.*.port` | 省略時はプロトコル標準ポート |
| `host.gateway: from_host` | 物理 NIC のデフォルト GW を自動検出 |
| `host.interface` | 例: `"Ethernet"` — 特定 NIC に限定 |

初回は **example のまま** で問題ありません。

### ステップ 4-5: 物理ゲートウェイの確認（`from_host` 利用時）

LAN に接続し、PC にデフォルトルートがある状態で実行します。

```powershell
.\scripts\lab.ps1 detect-gateway
```

**成功例:**

```json
{
  "gateway": "192.168.1.1",
  "interface": "Ethernet",
  "host_ip": "192.168.1.100",
  ...
}
```

| 結果 | 対処 |
|------|------|
| `gateway` が表示される | そのまま次へ |
| エラー / 空 | LAN ケーブル・IP 設定を確認。または `from_host` をやめて固定 IP で `gateway` を指定 |

検出結果は起動時に `generated\host-network.json` にも保存されます。

## 5. 起動

### ステップ 5-1: ラボの起動

プロジェクトルートで実行します。

```powershell
cd C:\testenv
.\scripts\lab.ps1 up
```

内部で次が自動実行されます。

1. `config\lab.json`（または yaml）の読み込み
2. 物理 GW の検出（設定で `from_host` の場合）
3. `docker-compose.yml` の生成
4. イメージのビルド（初回は数分かかることがあります）
5. コンテナの起動（`-d` バックグラウンド）

### ステップ 5-2: 状態確認

```powershell
.\scripts\lab.ps1 status
```

**期待される表示:** 対象コンテナの `STATE` が `running` であること。

```powershell
docker compose ps
```

## 6. 起動確認チェックリスト

以下を順に確認し、すべて ✓ なら起動完了です。

### 6-1. コンテナ一覧

```powershell
.\scripts\lab.ps1 status
```

- [ ] `lab-router` が `running`
- [ ] 有効にした各サービス（`lab-ntp-*` など）が `running`
- [ ] `Exited` や `Restarting` が無い

### 6-2. ルータとセグメント間通信

```powershell
docker exec lab-router ping -n 2 172.30.99.10
docker exec lab-router ping -n 2 172.30.10.53
```

- [ ] いずれも応答がある（`172.30.x.x` は設定に合わせて読み替え）

### 6-3. 生成ファイル

```powershell
dir generated
type generated\host-network.json
type generated\enabled-services.txt
```

- [ ] `host-network.json` に `gateway` が記録されている（`from_host` 使用時）
- [ ] `enabled-services.txt` に起動したサービス名が列挙されている

### 6-4. Syslog の受信テスト（任意）

```powershell
docker run --rm --network network-lab_segment-mgmt alpine sh -c "apk add -q netcat-openbsd && echo lab-test | nc -u -w1 172.30.99.14 514"
docker exec lab-syslog-main tail -n 3 /var/log/network-lab/messages.log
```

- [ ] ログに `lab-test` 相当の行が追加される

### 6-5. ログにエラーが無いこと

```powershell
docker compose logs router --tail 20
docker compose logs lab-dns-primary --tail 20
```

- [ ] 致命的な `Error` / `fatal` が無い

## 7. 日常運用コマンド

| 操作 | コマンド |
|------|----------|
| 起動 | `.\scripts\lab.ps1 up` |
| 停止 | `.\scripts\lab.ps1 down` |
| 再起動（設定反映） | `.\scripts\lab.ps1 restart` |
| compose のみ再生成 | `.\scripts\lab.ps1 generate` |
| ログ追跡 | `.\scripts\lab.ps1 logs` |
| GW 検出のみ | `.\scripts\lab.ps1 detect-gateway` |

**設定を変更したとき**は、必ず `restart`（または `down` → `up`）してください。

## 9. トラブルシューティング

| 現象 | 原因の例 | 対処 |
|------|----------|------|
| `Python 3 is required` | Python 未インストール | Python 3.9+ をインストールし PATH を通す |
| `Cannot read default gateway` | LAN 未接続・ルート無し | ケーブル／IP 設定を確認。`from_host` を外す |
| `port is already allocated` | ホストの 53/123 等が使用中 | `publish_ports: false` にする |
| ビルドが遅い／失敗 | 初回 DL・プロキシ | Docker Desktop を再起動。社内プロキシ設定を確認 |
| 設定変更が効かない | compose 未再生成 | `.\scripts\lab.ps1 restart` |
| `lab-router` だけ落ちる | 権限・sysctl | Docker Desktop を最新化。`docker compose logs router` を確認 |
| WSL2 関連エラー | 仮想化無効 | BIOS で仮想化を有効化し WSL2 を再インストール |

### 9.1 完全にやり直す場合

```powershell
.\scripts\lab.ps1 down
docker compose down -v
Remove-Item -Recurse -Force generated -ErrorAction SilentlyContinue
Remove-Item docker-compose.yml -ErrorAction SilentlyContinue
copy config\lab.json.example config\lab.json
.\scripts\lab.ps1 up
```

## 11. Git リポジトリへの登録（初回のみ）

管理者が GitHub / GitLab 等にリポジトリを作り、push すると他の PC から `git clone` できます。

### 11-1. ローカルで Git 初期化（未実施の場合）

```powershell
cd C:\testenv
git init
git add .
git status
git commit -m "Initial commit: network lab Docker environment"
```

`git status` で `config\lab.json` や `generated\` が **含まれていない** ことを確認してください（`.gitignore` 対象）。

### 11-2. リモートを追加して push

リポジトリ: https://github.com/okuson20-dev/testenv （空の状態で作成済みであること）

```powershell
git remote add origin https://github.com/okuson20-dev/testenv.git
git branch -M main
git push -u origin main
```

SSH の場合:

```powershell
git remote add origin git@github.com:okuson20-dev/testenv.git
git push -u origin main
```

### 11-3. 利用者側

```powershell
git clone https://github.com/okuson20-dev/testenv.git
```

以降は [セクション 4](#4-初回セットアップwindows) から続行します。
